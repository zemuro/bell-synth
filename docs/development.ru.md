## Как это работает

### Выбор окна затухания

Запись колокола начинается с короткого шумного удара языка. Анализатор пропускает первые `--attack-skip-ms` миллисекунд, чтобы работать только с затухающими обертонами. Если запрошенный пропуск длиннее самой записи, программа завершается с понятной ошибкой, а не молча выдаёт пустой спектр.

### Усреднение спектра

После пропуска атаки сигнал разбивается на перекрывающиеся кадры с окном Ханна. Для каждого кадра вычисляется магнитудный спектр через `numpy.fft.rfft`, после чего берётся среднее по всем кадрам. Это снижает случайный шум, сохраняя устойчивые спектральные особенности.

Размер FFT и шаг между кадрами задаются через `--fft-size` и `--hop-size`. Больший FFT даёт более точное частотное разрешение, но требует достаточной длины сигнала.

### Сглаживание

Усреднённый спектр сглаживается фильтром Савицкого-Голая. Это уменьшает высокочастотные флуктуации, не смещая положение пиков, и делает последующее обнаружение более устойчивым.

### Поиск пиков

Пики находятся с помощью `scipy.signal.find_peaks` в диапазоне `[min_freq, max_freq]`. Два параметра управляют чувствительностью:

- **prominence** — насколько пик выделяется на фоне окружения;
- **distance** — минимальное число бинов между соседними пиками.

Для негармонических спектров колоколов `distance` должен быть достаточно малым, чтобы различать близкие обертоны.

### Перевод частоты в ноту и центы

Каждая пиковая частота `f` независимо отображается на ближайший тон 12-тоновой равномерной темперации:

```text
semitones = 12 * log2(f / 440)
note_index = round(semitones)
deviation_cents = 1200 * log2(f / (440 * 2^(note_index / 12)))
```

Имя ноты выводится относительно A4 = 440 Гц; октава меняется на ноте C.

### Негармонические обертоны колоколов

В отличие от струнных или духовых инструментов, обертоны колоколов не кратны одной фундаментальной частоте. Поэтому каждый пик сопоставляется с 12-тоновой системой независимо, а не считается гармоникой. Именно поэтому для каждого обнаруженного пика выводится нота и отклонение в центах.

## Запуск тестов

```bash
python -m venv venv
venv\Scripts\pip install -r requirements.txt
venv\Scripts\python -m pytest tests/test_analyze_bell.py -v
```

Синтетические тестовые сэмплы генерируются автоматически во время выполнения тестов.

## Как добавить функциональность

1. Измените `analyze_bell.py`.
2. Если функция настраивается через конфиг, добавьте ключ в `CONFIG_SECTIONS` и `config_defaults()`.
3. Добавьте тесты в `tests/test_analyze_bell.py`.
4. При необходимости обновите `analyze_bell.ini`.
5. Обновите документацию на обоих языках.
6. Перед коммитом запустите полный набор тестов.

## Архитектурные замечания

- **Первый канал:** стереозаписи приводятся к каналу 0; многоканальный анализ вынесен в будущее.
- **Ленивый импорт matplotlib:** `matplotlib` и `pyplot` импортируются только при запросе визуализации, что ускоряет запуск в CSV-режиме.
- **Headless-режим:** при `--no-show` перед импортом `pyplot` вызывается `matplotlib.use("Agg")`, поэтому программа работает без дисплея.
- **Приоритет настроек:** CLI > конфиг > встроенные значения. Путь `--config` извлекается в первом проходе, чтобы argparse мог подставить значения из конфига как умолчания.

## Решение проблем

**Установка `soundfile` на Windows**
- Убедитесь, что есть рабочий C-компилятор, или используйте готовое колесо. Пакет основан на libsndfile.

**Чтение 24-битного PCM**
- `soundfile.read(..., dtype="float64")` корректно читает 24-битные WAV-файлы и возвращает нормализованные значения в `[-1.0, 1.0]`.

**Проблемы с бэкендом matplotlib**
- В headless-окружении используйте `--no-show` вместе с `--plot-save`.
- Бэкенд `Agg` устанавливается автоматически при наличии `--no-show`.
