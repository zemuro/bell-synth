# Руководство по использованию

Здесь описано, как запускать анализатор обертонов колокольных сэмплов из командной строки.

## Запуск

Анализатор принимает один позиционный аргумент — путь к WAV-файлу.

```bash
python analyze_bell.py samples/bell.wav
```

Если файл не указан, по умолчанию используется `samples\bell.wav` относительно корня проекта.

## Справочник по флагам

| Аргумент | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `input` | путь | `samples\bell.wav` | Входной WAV-файл |
| `--output` | путь | — | Путь для CSV/таблицы; без него вывод идёт в stdout |
| `--attack-skip-ms` | float | `100.0` | Длительность пропуска в начале, мс |
| `--min-freq` | float | `50.0` | Минимальная частота пика, Гц |
| `--max-freq` | float | `8000.0` | Максимальная частота пика, Гц |
| `--prominence` | float | `0.005` | Минимальная выраженность пика |
| `--distance` | int | `20` | Минимальное число бинов между пиками |
| `--smoothing-window` | int | `11` | Длина окна сглаживания, нечётное ≥ 3 |
| `--fft-size` | int | `16384` | Размер FFT на кадр |
| `--hop-size` | int | `2048` | Шаг между кадрами |
| `--peak-count` | int | — | Максимальное число пиков; пустое значение — без ограничения |
| `--format` | выбор | `csv` | Формат вывода: `csv` или `table` |
| `--visualize` | флаг | — | Открыть интерактивное окно matplotlib |
| `--spectrogram` | флаг | — | То же, что `--visualize` |
| `--plot-save` | путь | — | Сохранить график в PNG |
| `--no-show` | флаг | — | Не открывать интерактивное окно |
| `--quiet` | флаг | — | Подавить текстовый вывод |
| `--spec-nperseg` | int | `4096` | Длина окна STFT для спектрограммы |
| `--spec-noverlap` | int | `3072` | Перекрытие STFT |
| `--spec-nfft` | int | `4096` | Длина FFT для STFT |
| `--spectrum-floor` | float | `-50.0` | Нижняя граница графика спектра, дБ |
| `--spec-floor` | float | `-144.0` | Нижняя граница цветовой шкалы спектрограммы, дБ |
| `--config`, `-c` | путь | — | Путь к пользовательскому INI-файлу |
| `--save-config` | путь | — | Записать эффективную конфигурацию и завершиться |
| `--n-labels`, `-n` | int | `7` | Число подписываемых пиков на графике спектра |

## Приоритет настроек

Значения выбираются в таком порядке:

1. **Флаги CLI** — высший приоритет.
2. **Конфигурационный файл** (`analyze_bell.ini` или файл из `--config`).
3. **Встроенные значения по умолчанию** — низший приоритет.

## Примеры

### Базовый CSV-вывод

```bash
python analyze_bell.py samples/bell.wav
```

### Табличный вывод с пользовательным диапазоном

```bash
python analyze_bell.py samples/bell.wav --min-freq 200 --max-freq 4000 --format table
```

### Интерактивная визуализация

```bash
python analyze_bell.py samples/bell.wav --visualize --peak-count 12
```

### Headless-экспорт PNG

```bash
python analyze_bell.py samples/bell.wav --plot-save bell_report.png --no-show
```

### Загрузка пользовательского конфига

```bash
python analyze_bell.py samples/bell.wav --config my_bell.ini
```

### Сохранение конфигурации

```bash
python analyze_bell.py --save-config my_bell.ini
```

### Переопределение значения из командной строки

```bash
python analyze_bell.py samples/bell.wav --config my_bell.ini --min-freq 200.0
```

## Советы по визуализации

- Чтобы график не был перегружен метками, уменьшайте `--n-labels`. По умолчанию подписываются 7 самых сильных пиков.
- На сервере или в CI всегда используйте `--no-show` вместе с `--plot-save`.
- Если тихие пики обрезаются, понизьте `--spectrum-floor`; если график выглядит пустым — повысьте.
- Если спектрограмма слишком шумная или, наоборот, тёмная, подберите `--spec-floor`.

## Устранение неполадок

**Не находит пики**
- Уменьшите `--prominence`, например до `0.001`.
- Уменьшите `--distance`, например до `10`, чтобы различить близкие обертоны.
- Расширьте диапазон `--min-freq` / `--max-freq`.

**Слишком большой пропуск атаки**
- Уменьшите `--attack-skip-ms` или возьмите более длинный сэмпл.

**Метки перекрываются**
- Уменьшите `--n-labels`.

**PNG пустой**
- Проверьте, что `--plot-save` получил корректный путь.
- В headless-окружении обязательно добавьте `--no-show`.

**Не найден конфигурационный файл**
- Проверьте путь, переданный в `--config`.
- Используйте `--save-config`, чтобы сгенерировать корректный стартовый файл.
